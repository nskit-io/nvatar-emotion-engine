# NVatar Emotion Engine

**9차원 감정 추적 + MBTI 연속 스펙트럼 + decay/commit 성격 진화 + 3단계 기억 압축 + 사용자 성격 추론 — AI 캐릭터를 위한 통합 인지 엔진.**

> [NVatar](https://github.com/nskit-io/nvatar) 프로젝트에서 추출 — 로컬 하드웨어에서 완전히 구동되는 AI 아바타 채팅 시스템.

---

## 왜 만들었나

오늘날의 LLM 캐릭터는 상태가 없습니다. 잊어버리고, 성장하지 않고, 감정도 없습니다. 메모리 라이브러리는 존재하지만(Letta, Mem0), **감정 + 성격 진화 + 기억**을 하나의 통합 시스템으로 다루는 것은 없습니다.

이 엔진이 AI 캐릭터에게 부여하는 것:
- **감정**: 대화 중 자연스럽게 변동하고 기준선으로 감쇄 (호기심 포함 9차원)
- **MBTI 스펙트럼**: 연속적 0-100 스케일에서 확률적 표현으로 진화 — 고정된 유형이 아님
- **성격**: 주 단위 상호작용으로 진화 — 독창적인 decay/commit 메커니즘
- **기억**: 원본 메시지 → 요약 → 키워드로 압축, 사람의 기억이 흐려지는 것처럼
- **사용자 이해** — 아바타가 대화 패턴에서 당신의 성격을 추론하고 스타일을 조정

## 아키텍처

```
사용자 메시지
    │
    ├─→ 감정 서비스 ──────→ 9차원 감정 벡터 업데이트
    │                        (키워드 트리거, 자연 감쇄)
    │
    ├─→ 기억 서비스 ──────→ L1 원본 → L2 이벤트 → L3 키워드
    │                        (자동 압축 + L2→L3 자동 전환)
    │
    ├─→ 성격 진화 ────────→ 특성 드리프트 + decay/commit
    │                        (미결 델타 → 시간 감쇄 → 커밋)
    │
    ├─→ MBTI 분석기 ──────→ 연속 스펙트럼 진화
    │                        (블라인드 분석 → 델타 → 가중 적용)
    │
    └─→ 사용자 추론 ──────→ 사용자 메시지에서 성격 추론
                             (압축 사이클에 걸쳐 축적)
```

## 감정 시스템 — 9차원

감성 분석이 아닙니다. 분류도 아닙니다. 대화를 넘어 **지속되는 연속적 감정 상태**입니다.

| 차원 | 경향 |
|------|------|
| 기쁨 | 긍정적 상호작용으로 상승 |
| 슬픔 | 상실/실망 주제에 반응 |
| 분노 | 갈등 시 급등, 가장 빠르게 감쇄 |
| 불안 | 불확실성에 축적 |
| 애정 | 신뢰와 함께 천천히 성장 |
| 흥분 | 새로운 것에 급등 |
| 지루함 | 반복에 상승 |
| 신뢰 | 가장 느리게 변하는 차원 |
| **호기심** | **새로운 주제에 상승 — 후속 질문과 깊은 탐구를 유발** |

**메커니즘:**
- 키워드가 감정 변화를 유발
- 여러 턴에 걸쳐 기준선으로 자연 감쇄
- 유의미하게 높아진 감정만 프롬프트에 반영
- 각 메시지에 `emotion_snapshot` 저장

## 성격 진화 — Decay/Commit

**이 메커니즘의 오픈소스 선례는 없습니다.**

8개 성격 특성이 3단계 파이프라인으로 진화:

### 3단계 파이프라인

**1단계 — 미결 델타 축적**
각 메시지가 특성 델타를 생성 — 설정 가능한 상한으로 극단적 변동 방지.

**2단계 — 시간 감쇄**
마지막 대화 이후 시간에 따른 지수 감쇄 — 최근 대화는 강하게, 오래된 대화는 자연스럽게 사라짐.

**3단계 — 압축 커밋**
미결 델타에 감쇄 계수를 적용하여 특성 값에 커밋, 커밋당 설정 가능한 상한 적용.

**저항 곡선:** 극단적 값은 추가 변화에 저항 — 실제 사람처럼.

모든 커밋은 분석을 위해 로깅됩니다.

## MBTI 연속 스펙트럼

고정된 성격 유형(INTJ, ENFP...)과 달리, NVatar는 4개 MBTI 차원에 걸쳐 **연속적 0-100 스펙트럼**을 사용합니다. 아바타는 하나의 유형에 고정되지 않습니다 — 대화를 통해 변화하는 살아있는 성격입니다.

| 차원 | 0 | 50 (경계) | 100 |
|------|---|-----------|-----|
| E/I | 깊은 내향형 | 양향형 | 깊은 외향형 |
| S/N | 강한 감각형 | 균형 | 강한 직관형 |
| T/F | 강한 사고형 | 균형 | 강한 감정형 |
| J/P | 강한 판단형 | 균형 | 강한 인식형 |

### 확률적 표현

T=70이더라도 아바타는 ~30%의 확률로 공감적 반응을 보입니다. 이는 성격의 경직성을 방지하고 자연스럽고 인간적인 변동을 만들어냅니다.

### 3단계 MBTI 분석 파이프라인

MBTI 진화는 기억 압축 시 발생합니다:

1. **블라인드 분석** — LLM이 현재 점수를 모른 채 아바타의 최근 응답을 분석
2. **델타 계산** — 블라인드 결과와 현재 스펙트럼 간의 차이
3. **가중 적용** — L1→L2 및 L2→L3 전환 시 설정 가능한 가중치로 델타 적용, 극단값에서 저항

## 사용자 성격 추론

아바타는 자신의 성격만 진화시키는 것이 아닙니다 — **당신의 성격도 학습합니다**.

각 압축 사이클마다 사용자 메시지를 별도로 분석하여 MBTI 성향을 추론합니다. 이는 여러 사이클에 걸쳐 축적됩니다:

| 신뢰도 | 아바타 행동 |
|--------|------------|
| 초기 (1-2회 분석) | 미지 — 다양한 질문을 통해 당신에 대해 학습 |
| 부상 (2-3회 분석) | 미묘한 적응 — 언급하지 않고 소통 스타일 조정 |
| 확립 (3회+ 분석) | 명시적 개인화 — "직접적인 답변을 선호하시는 걸 알기에..." |

추론된 사용자 성격은 아바타의 시스템 프롬프트에 주입되어, 주 단위 상호작용을 통해 점점 개선되는 진정한 맞춤형 대화를 가능하게 합니다.

## 기억 시스템 — 3단계

```
L1 (원본)   → 최근 메시지를 프롬프트에 포함
    ↓ 압축
L2 (이벤트) → 관련도 점수 포함 요약
    ↓ 낮은 관련도
L3 (키워드) → 장기 단편, 불확실하게 언급
```

- **L1**: 설정 가능한 메시지 한도와 자동 압축 트리거
- **L2**: 관련도 점수는 시간이 지나면 감쇄; 재언급 시 관련도 상승; 낮은 관련도의 이벤트는 L3로 이동
- **L3**: 키워드로 추출. "기억이 잘 안 나는데..." 불확실하게 발화
